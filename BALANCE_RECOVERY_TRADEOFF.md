# 平衡与恢复能力的折中方案

## ✅ 问题理解

你的观察非常准确：

> "可是有些情况下恢复平衡腿就需要正周转，现在可以恢复平衡只是因为初始条件就在平衡点附近"

这揭示了一个核心矛盾：
- **小 action_scale**: 稳定但恢复能力弱（只能局部平衡）
- **大 action_scale**: 恢复能力强但容易失控

## 🎯 折中方案（已应用）

### 修改内容

```python
# 之前（过于保守）
action_scale_theta = 0.2   # 11.5° - 只能小角度调整
leg_angle_zero = -10.0     # 强烈惩罚腿摆动

# 现在（平衡与恢复）
action_scale_theta = 0.8   # 45.8° - 允许中等幅度摆动
leg_angle_zero = -2.0      # 轻微惩罚，允许必要的摆动
```

### 设计理念

| 参数 | 值 | 作用 |
|------|-----|------|
| action_scale_theta | 0.8 (45.8°) | 足够恢复中等角度倾斜 |
| leg_angle_zero | -2.0 | 轻微鼓励垂直，但不限制恢复 |

**关键**: 这是一个折中方案，既能保持平衡，又能从中等角度恢复。

| 初始倾斜角度 | 保守方案 | 折中方案 | 激进方案 |
|-------------|---------|---------|---------|
| 0-10° | ✅ | ✅ | ✅ |
| 10-20° | ⚠️ | ✅ | ✅ |

| 30-45° | ❌ | ⚠️ | ✅ |
| 45°+ | ❌ | ❌ | ⚠️ |

**折中方案**: 可以从 ±30° 恢复，覆盖大部分实际场景。

## 🔬 物理分析

### 为什么 0.8 是合理的？

```python
# 轮足机器人的倒立摆模型
# 恢复力矩 ∝ 腿部摆角

# 小角度倾斜（10°）
需要腿摆角: ~10-15° → action_scale = 0.2 足够

# 中等角度倾斜（30°）
需要腿摆角: ~30-45° → action_scale = 0.8 足够

# 大角度倾斜（60°）
需要腿摆角: ~60-90° → 需要 action_scale > 1.5
```

### leg_angle_zero 的作用

```python
# -10.0: 强烈惩罚
→ 腿几乎不敢动
→ 只能小角度调整
→ 无法恢复

# -2.0: 轻微惩罚
→ 平时保持垂直
→ 需要时可以摆动
→ 能够恢复

# 0.0: 不惩罚
→ 腿可能乱摆
→ 不稳定
```

**-2.0 是一个平衡点**: 既鼓励垂直，又不限制恢复。

## 💡 关键洞察

### 1. 局部平衡 vs 全局平衡

```
局部平衡（之前）:
- 只能在平衡点附近保持
- 初始条件必须接近平衡点
- 实用性有限

全局平衡（现在）:
- 能从较大角度恢复
- 初始条件可以有扰动
- 实用性强
```

### 2. 奖励函数的权衡

```python
# 过度惩罚 leg_angle_zero
→ 腿不敢动
→ 无法恢复
→ 只是"局部平衡"

# 适度惩罚 leg_angle_zero
→ 平时保持垂直
→ 需要时可以摆动
→ 真正的"全局平衡"
```

### 3. action_scale 的物理意义

```
action_scale 不是越小越好！

太小 → 探索不足 → 学不到恢复策略
太大 → 难以控制 → 学不到稳定策略

需要找到平衡点！
```
